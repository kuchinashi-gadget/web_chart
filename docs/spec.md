# Chart Playback 仕様メモ

## 目的

Chart Playback は、過去チャートを1本ずつ進めながら、売買判断・建玉管理・振り返りを練習する Web アプリ。

投資助言ではなく、チャート読解と売買判断を練習するための個人学習ツール。

## 2026-07-02 時点の方針

- Webアプリ名、公式サイト名ともに `チャートプレイバック` に統一する。
- 英語表記は `Chart Playback` とする。
- 無料会員の保存済みCSV銘柄数は **10銘柄まで** とする。
- ゲストはCSV銘柄追加不可とし、架空サンプル銘柄で主要機能を体験できるようにする。
- Pro / 有料会員ではクラウド保存、真剣練習モード、練習結果サマリーを提供する候補とする。
- プロモーションビデオは公式サイト最小版の作成後、ログイン実装より前に作る。


## 2026-07-02 時点の到達点

株価データの CSV 読み込みは実装済み。

現在は、ツールバーの `CSV読込` / `Import CSV` からユーザーが CSV を選択し、一時銘柄としてチャート表示できる。
さらに、読み込んだ CSV を保存済み銘柄として登録し、リロード後も銘柄選択から再表示できる。

CSV データ本体は IndexedDB、銘柄メタ情報は localStorage に保存する。
保存済み銘柄は、設定ダイアログの「銘柄管理」タブから編集・CSV差し替え・削除できる。

## 対象データ

現在は、CSV を `public/data` に配置して読み込む方法に加えて、ユーザーがブラウザから CSV ファイルを選択してチャートに読み込む方法にも対応している。

基本列:

```csv
Date,Open,High,Low,Close,Volume
```

必須列:

- Date
- Open
- High
- Low
- Close

任意列:

- Volume

Volume は未使用でもよい。

対応候補:

- 日本株
- 指数
- 米国株
- 先物
- 為替
- 仮想通貨
- ETF

銘柄定義には以下を持つ。

- code
- name
- path
- currency
- unitLabel
- defaultLotSize
- multiplier

ユーザーが読み込んだ CSV については、以下の銘柄情報を設定できるようにする。

- 銘柄名
- コード
- 市場 / 種別
- 通貨
- 1玉あたり数量
- multiplier
- 表示名

## ユーザー CSV 読み込み

段階的に実装する。

### ステップ1: 一時読み込み

実装済み。

- ファイル選択で CSV を読み込む。
- `Date,Open,High,Low,Close` を検証する。
- `Volume` は任意。
- 読み込めたら一時銘柄としてチャート表示する。
- 一時銘柄のままでは、リロードしたら消えてもよい。
- 保存済み銘柄として登録すれば、リロード後も利用できる。

現在の挙動:

- ツールバーの `CSV読込` / `Import CSV` から CSV を選択する。
- ファイル名からコード・表示名を仮決定する。
- 価格の小数桁は CSV 内の `Open,High,Low,Close` から自動推定する。
- 通貨、1玉あたり数量、倍率などは仮の汎用設定で読み込む。
- 一時銘柄のままでは、読み込んだ CSV はブラウザ内のメモリ上だけに保持し、リロードすると消える。
- 銘柄一覧に保存した場合は、CSV データ本体を IndexedDB に保存し、リロード後も再表示できる。

### ステップ2: 銘柄情報設定

実装済み。

読み込んだ CSV に対して、以下を設定できるようにする。

- 銘柄名
- コード
- 市場 / 種別
- 通貨
- 1玉あたり数量
- multiplier

現在の挙動:

- CSV 読み込み直後に一時銘柄設定ダイアログを表示する。
- 一時銘柄を選択中は、ツールバーの歯車ボタンから再編集できる。
- 設定できる項目は、コード、銘柄名、通貨、単位、1玉あたり数量、倍率、価格小数桁。
- 保存すると銘柄表示、数量表示、損益計算、価格小数桁に即反映する。
- 一時銘柄のままでは、設定は一時読み込み中のみ有効で、リロードすると消える。
- 銘柄一覧に保存した場合は、銘柄メタ情報として保持される。

### ステップ3: 銘柄管理

- 読み込んだ CSV を銘柄一覧に追加する。実装済み。
- 銘柄一覧から選択できる。実装済み。
- 名前変更できる。実装済み。
- 削除できる。実装済み。
- 再インポートできる。実装済み。

CSV データ本体は大きくなる可能性があるため、保存する場合は localStorage ではなく IndexedDB を優先する。

現在の保存仕様:

- 一時CSV銘柄の設定画面から「銘柄一覧に保存」する。
- CSV データ本体は IndexedDB に保存する。
- 銘柄メタ情報は localStorage に保存する。
- 保存済み銘柄は、リロード後も銘柄選択プルダウンに表示される。
- 一時銘柄で作成した売買練習データとチャートメモは、保存済み銘柄側へ引き継ぐ。
- 保存済み銘柄を選択中は、ツールバーの歯車ボタンから設定を再編集できる。
- 保存済み銘柄の CSV を差し替えると、同じ銘柄の価格データを更新する。
- 保存済み銘柄を削除すると、その銘柄の売買練習データとチャートメモも削除する。
- 設定ダイアログの「銘柄管理」タブから、保存済み銘柄を一覧管理できる。
- 銘柄管理タブでは、編集、CSV差し替え、削除、選択中表示、売買ログ数、チャートメモ数を確認できる。
- 一時CSV銘柄を保存する際、同じコードの保存済み銘柄がある場合は警告する。
- 重複時は、既存銘柄への上書き、別銘柄として保存、キャンセルを選べる。

将来のアカウント別制限:

- ゲスト: CSV銘柄追加は不可。架空サンプル銘柄のみ利用可能にする想定。
- 無料会員: CSV銘柄追加と銘柄管理が可能。保存できる銘柄数を10件までにする。
- 有料会員: 保存できる銘柄数を拡張または無制限にする想定。
- 並べ替え、一括削除、非表示、保存済み銘柄一覧の検索など高度な銘柄管理は有料向け候補。

正式公開時のURL構成候補:


現在の `https://web-chart.stock-practice.workers.dev/` は仮公開URLとして扱い、正式公開前に変更してよい。

## ゲスト用サンプルデータ

ゲストは、CSV銘柄追加をできない方向で検討する。

その代わり、主要機能を試せる架空サンプル銘柄を用意する。

サンプルデータ方針:

- 実在企業名を使わない。
- 実在ティッカーを使わない。
- 実在株価をそのまま使わない。
- 実在株価を少し加工しただけのデータも避ける。
- Pythonなどで完全に生成した架空OHLCデータにする。
- 5〜8銘柄程度から始める。

候補:

```text
DEMO-AUTO   架空自動車: 上昇トレンド
DEMO-TECH   架空テック: 急騰・急落あり
DEMO-FOOD   架空食品: 安定した値動き
DEMO-BANK   架空金融: レンジ相場
DEMO-BIO    架空バイオ: 高ボラティリティ
DEMO-FX     架空ドル円風: 小数価格・為替風
```

注意書き:

```text
サンプル銘柄はすべて架空データです。
実在の企業・銘柄・指数・為替レートとは関係ありません。
投資判断には使用できません。
```

ドル円風サンプルを置く場合も、実際のUSD/JPYではないことを明記する。

## チャート機能

- ローソク足表示。
- 移動平均線表示。
- 日足 / 週足 / 月足の切り替え。
- キーボード左右キー、画面ボタンで1本送り。
- マウスホイールや横スクロールで表示位置を移動。
- 表示本数は以下から選択。
  - 毎回自動
  - 前回を引き継ぐ
  - 固定本数
- 初期表示位置は以下から選択。
  - 最新
  - 最新からN本前
  - 前回の続き
  - 最初の方
- 表示本数と右端日付は銘柄・足種ごとに保存。

## 足種切り替えの日付仕様

日足から週足・月足へ切り替える時は、基準日より未来の未完成足を表示しない。

例:

- 日足が 2020/6/4 の場合
  - 週足は 2020/5/29 まで
  - 月足は 2020/5/29 まで
- 日足へ戻した時は、元の日足 2020/6/4 に戻る。

週足・月足側で移動した場合は、その足に含まれる最後の日足へ基準日を更新する。

## 売買練習

注文種別:

- 売りを追加
- 売りを返済
- 買いを追加
- 買いを返済

建玉表示:

```text
売玉 - 買玉
```

売買は FIFO で返済する。

約定タイミング:

- 次の取引日の始値
- 当日の終値

設定から選択できる。

通常練習では、過去に戻れる・銘柄を変えられる・ログ削除できる。

そのため、練習結果サマリーは通常練習では急いで実装しない。

## 真剣練習モード

有料版 / Pro 版向けの候補。

通常練習より厳密な練習として扱う。

制限候補:

- 時間を戻せない。
- 途中で銘柄を変えられない。
- 売買ログを削除できない。
- 練習途中で条件を変更しにくくする。
- 練習終了時に結果サマリーを表示する。
- 練習履歴を保存する。

練習結果サマリーは、この真剣練習モードとセットで実装する方針。

通常練習は気軽に練習するための機能、真剣練習は成績を残すための機能として分ける。

- 勝ち数

- 引き分け数
- 総損益

- 最大損失
- 売買回数
- 練習期間
- 最大ドローダウン
- 平均利益
- 平均損失
- 勝率
- 損益率

## 損益計算

損益は以下で計算する。

```text
価格差 × 1玉あたり数量 × 玉数 × multiplier
```

日本株は基本的に:

```text
1玉 = 100株
currency = JPY
multiplier = 1
```

指数などは:

```text
1玉 = 1
unitLabel = ""
currency = JPY
multiplier = 1
```

## 売買ログ

売買ログには以下を表示する。

- 注文種別
- 玉数
- 数量
- 注文日
- 約定日
- 約定価格
- 確定損益
- 返済した建玉
- 勝敗

建玉ログと返済ログは hover / click で対応関係をハイライトする。

削除仕様:

- 返済ログを削除した場合、その返済で閉じた建玉を復活する。
- 建玉ログを削除した場合、紐づく返済ログも一緒に削除する。
- 削除対象の返済ログに別の建玉が混ざっていた場合、その別建玉は復活する。

真剣練習モードでは、売買ログ削除をできないようにする。

## 勝敗集計

返済ログを対象に勝敗を集計する。

- 損益率が引き分け基準未満: 引き分け
- それ以上の利益: 勝ち
- それ以上の損失: 負け

初期の引き分け基準:

```text
0.5%
```

設定画面から変更できる。

勝敗は損益と同じように表示/非表示を切り替えられる。

- 今後、メモ色や追加 UI をさらに改善予定。

## 本数計測

チャート上でローソク足の本数を数える機能。

操作:

1. 「本数計測」ボタン、または `C` キーで本数計測モードを ON。
2. 始点のローソク足をクリック。
3. 終点のローソク足をクリック。
4. 選択範囲と本数を表示。

補足:

- 本数は始点と終点を含めて数える。
- 3回目のクリックで新しい計測を開始する。
- `Esc` で計測をクリアする。
- 足種変更・銘柄変更時は計測をクリアする。

## ファイル分割方針

2026-07-02 時点の `App.tsx` は約8600行・約300KBあり、今後の保守やCodex作業の安全性を考えると分割が必要。

ただし、UIコンポーネントまで一気に分ける大規模リファクタは避ける。
まずは、依存が比較的分かりやすい定数・ユーティリティ・保存処理から小さく分ける。

実施済み:

- 共通型を `src/types.ts` に分割。
- 既定銘柄定義を `src/data/defaultSymbols.ts` に分割。
- CSV から日足ローソク足へ変換する処理を `src/utils/csv.ts` に分割。

### 推奨する分割順

#### 1. `src/constants/storageKeys.ts`

storage key と IndexedDB 名を分離する。

対象:

- `TRADING_BOOKS_STORAGE_KEY`
- `CHART_SETTINGS_STORAGE_KEY`
- `CHART_VIEW_STATE_STORAGE_KEY`
- `SOUND_ENABLED_STORAGE_KEY`
- `PAINT_MARKS_STORAGE_KEY`
- `PAINT_CUSTOM_COLORS_STORAGE_KEY`
- `PAINT_TOOL_COLORS_STORAGE_KEY`
- `PAINT_TEXT_SETTINGS_STORAGE_KEY`
- `PAINT_PRACTICE_DB_NAME`
- `PAINT_PRACTICE_STORE_NAME`
- `USER_SYMBOLS_STORAGE_KEY`
- `USER_SYMBOLS_DB_NAME`
- `USER_SYMBOLS_STORE_NAME`

#### 2. `src/utils/date.ts`
