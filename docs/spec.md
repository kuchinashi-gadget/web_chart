# Chart Playback 仕様メモ

## 目的

Chart Playback は、過去チャートを1本ずつ進めながら、売買判断・建玉管理・振り返りを練習する Web アプリ。

投資助言ではなく、チャート読解と売買判断を練習するための個人学習ツール。

## 対象データ

現在は、CSV を `public/data` に配置して読み込む。

今後は、ユーザーがブラウザから CSV ファイルを選択し、チャートに読み込めるようにする。

基本列:

```csv
Date,Open,High,Low,Close,Volume
```

必須列:

- Date
- Open
- High
- Low
- Close

任意列:

- Volume

Volume は未使用でもよい。

対応候補:

- 日本株
- 指数
- 米国株
- 先物
- 為替
- 仮想通貨
- ETF

銘柄定義には以下を持つ。

- code
- name
- path
- currency
- unitLabel
- defaultLotSize
- multiplier

ユーザーが読み込んだ CSV については、以下の銘柄情報を設定できるようにする。

- 銘柄名
- コード
- 市場 / 種別
- 通貨
- 1玉あたり数量
- multiplier
- 表示名

## ユーザー CSV 読み込み

段階的に実装する。

### ステップ1: 一時読み込み

実装済み。

- ファイル選択で CSV を読み込む。
- `Date,Open,High,Low,Close` を検証する。
- `Volume` は任意。
- 読み込めたら一時銘柄としてチャート表示する。
- この段階では、リロードしたら消えてもよい。

現在の挙動:

- ツールバーの `CSV読込` / `Import CSV` から CSV を選択する。
- ファイル名からコード・表示名を仮決定する。
- 価格の小数桁は CSV 内の `Open,High,Low,Close` から自動推定する。
- 通貨、1玉あたり数量、倍率などは仮の汎用設定で読み込む。
- 読み込んだ CSV はブラウザ内のメモリ上だけに保持し、リロードすると消える。

### ステップ2: 銘柄情報設定

実装済み。

読み込んだ CSV に対して、以下を設定できるようにする。

- 銘柄名
- コード
- 市場 / 種別
- 通貨
- 1玉あたり数量
- multiplier

現在の挙動:

- CSV 読み込み直後に一時銘柄設定ダイアログを表示する。
- 一時銘柄を選択中は、ツールバーの歯車ボタンから再編集できる。
- 設定できる項目は、コード、銘柄名、通貨、単位、1玉あたり数量、倍率、価格小数桁。
- 保存すると銘柄表示、数量表示、損益計算、価格小数桁に即反映する。
- 設定は一時読み込み中のみ有効で、リロードすると消える。

### ステップ3: 銘柄管理

- 読み込んだ CSV を銘柄一覧に追加する。実装済み。
- 銘柄一覧から選択できる。実装済み。
- 名前変更できる。未実装。
- 削除できる。未実装。
- 再インポートできる。未実装。

CSV データ本体は大きくなる可能性があるため、保存する場合は localStorage ではなく IndexedDB を優先する。

現在の保存仕様:

- 一時CSV銘柄の設定画面から「銘柄一覧に保存」する。
- CSV データ本体は IndexedDB に保存する。
- 銘柄メタ情報は localStorage に保存する。
- 保存済み銘柄は、リロード後も銘柄選択プルダウンに表示される。
- 一時銘柄で作成した売買練習データとチャートメモは、保存済み銘柄側へ引き継ぐ。

## チャート機能

- ローソク足表示。
- 移動平均線表示。
- 日足 / 週足 / 月足の切り替え。
- キーボード左右キー、画面ボタンで1本送り。
- マウスホイールや横スクロールで表示位置を移動。
- 表示本数は以下から選択。
  - 毎回自動
  - 前回を引き継ぐ
  - 固定本数
- 初期表示位置は以下から選択。
  - 最新
  - 最新からN本前
  - 前回の続き
  - 最初の方
- 表示本数と右端日付は銘柄・足種ごとに保存。

## 足種切り替えの日付仕様

日足から週足・月足へ切り替える時は、基準日より未来の未完成足を表示しない。

例:

- 日足が 2020/6/4 の場合
  - 週足は 2020/5/29 まで
  - 月足は 2020/5/29 まで
- 日足へ戻した時は、元の日足 2020/6/4 に戻る。

週足・月足側で移動した場合は、その足に含まれる最後の日足へ基準日を更新する。

## 売買練習

注文種別:

- 売りを追加
- 売りを返済
- 買いを追加
- 買いを返済

建玉表示:

```text
売玉 - 買玉
```

売買は FIFO で返済する。

約定タイミング:

- 次の取引日の始値
- 当日の終値

設定から選択できる。

通常練習では、過去に戻れる・銘柄を変えられる・ログ削除できる。

そのため、練習結果サマリーは通常練習では急いで実装しない。

## 真剣練習モード

有料版 / Pro 版向けの候補。

通常練習より厳密な練習として扱う。

制限候補:

- 時間を戻せない。
- 途中で銘柄を変えられない。
- 売買ログを削除できない。
- 練習途中で条件を変更しにくくする。
- 練習終了時に結果サマリーを表示する。
- 練習履歴を保存する。

練習結果サマリーは、この真剣練習モードとセットで実装する方針。

通常練習は気軽に練習するための機能、真剣練習は成績を残すための機能として分ける。

## 練習結果サマリー

通常練習では保留。

真剣練習モードで実装する。

表示候補:

- 勝ち数
- 負け数
- 引き分け数
- 総損益
- 最大利益
- 最大損失
- 売買回数
- 練習期間
- 最大ドローダウン
- 平均利益
- 平均損失
- 勝率
- 損益率

## 損益計算

損益は以下で計算する。

```text
価格差 × 1玉あたり数量 × 玉数 × multiplier
```

日本株は基本的に:

```text
1玉 = 100株
currency = JPY
multiplier = 1
```

指数などは:

```text
1玉 = 1
unitLabel = ""
currency = JPY
multiplier = 1
```

## 売買ログ

売買ログには以下を表示する。

- 注文種別
- 玉数
- 数量
- 注文日
- 約定日
- 約定価格
- 確定損益
- 返済した建玉
- 勝敗

建玉ログと返済ログは hover / click で対応関係をハイライトする。

削除仕様:

- 返済ログを削除した場合、その返済で閉じた建玉を復活する。
- 建玉ログを削除した場合、紐づく返済ログも一緒に削除する。
- 削除対象の返済ログに別の建玉が混ざっていた場合、その別建玉は復活する。

真剣練習モードでは、売買ログ削除をできないようにする。

## 勝敗集計

返済ログを対象に勝敗を集計する。

- 損益率が引き分け基準未満: 引き分け
- それ以上の利益: 勝ち
- それ以上の損失: 負け

初期の引き分け基準:

```text
0.5%
```

設定画面から変更できる。

勝敗は損益と同じように表示/非表示を切り替えられる。

## チャートメモ

- チャート上のローソク足をクリックして対象日を選べる。
- メモはクリックした日付に紐づく。
- 文章だけのメモも表示される。
- 表示外にあるメモの対象日は、表示範囲の左端または右端に寄せて扱う。
- 今後、メモ色や追加 UI をさらに改善予定。

## ペイント練習

ライブチャートに直接描画するのではなく、チャートをスクリーンショット化し、その画像に描画する。

現在できること:

- 現在のチャートをスクショして開始。
- 直線、曲線、フリーハンド、矢印、丸・楕円、四角、テキスト、消しゴム。
- 色選択、カスタム色。
- 太さ変更。
- PNG ダウンロード。
- ペイント履歴保存。
- 元に戻す / やり直し。
- `Ctrl+Z`、`Ctrl+Y`、`Ctrl+Shift+Z`。
- テキスト入力欄をクリック位置に表示。
- テキスト入力欄へ自動フォーカス。
- テキストサイズ・フォント選択。
- テキスト色保持。
- 追加後のテキスト移動。

## 本数計測

チャート上でローソク足の本数を数える機能。

操作:

1. 「本数計測」ボタン、または `C` キーで本数計測モードを ON。
2. 始点のローソク足をクリック。
3. 終点のローソク足をクリック。
4. 選択範囲と本数を表示。

補足:

- 本数は始点と終点を含めて数える。
- 3回目のクリックで新しい計測を開始する。
- `Esc` で計測をクリアする。
- 足種変更・銘柄変更時は計測をクリアする。

## 外観・設定

設定画面の主な項目:

- 移動平均線
- 表示
- チャート外観
- 売買設定
- 言語

言語:

- 日本語
- 英語

外観はチャートだけではなく、アプリ全体にテーマが反映される方向で調整中。

## 保存

ブラウザの localStorage / IndexedDB を使う。

保存対象:

- 売買練習データ
- チャートメモ
- チャート設定
- 表示本数・右端日付
- ペイント履歴
- ユーザーが読み込んだ銘柄メタ情報
- ユーザーが読み込んだ CSV データ

CSV データ本体は IndexedDB を優先する。

設定や小さなメタ情報は localStorage でもよい。

クラウド保存やユーザー登録は後回し。

## ファイル分割方針

CSV 読み込み機能と銘柄管理機能を追加する前に、関係する部分だけを小さく分割する。

最初に分割する候補:

- `src/types.ts`
- `src/data/defaultSymbols.ts`
- `src/utils/csv.ts`
- `src/utils/storage.ts`

実施済み:

- 共通型を `src/types.ts` に分割。
- 既定銘柄定義を `src/data/defaultSymbols.ts` に分割。
- CSV から日足ローソク足へ変換する処理を `src/utils/csv.ts` に分割。

保留:

- `src/utils/storage.ts`

保存処理は App 内の型や状態への依存が広いため、次回以降に小さく分ける。

この段階では、UI変更や新機能追加はしない。

大規模な App.tsx 全体リファクタは後回し。

候補:

## 注意

- 投資助言ではない。
- 損益計算は簡易計算。
- 手数料、税金、スプレッド、証拠金、先物や為替の取引単位などを完全には反映していない。
- 市場データの正確性、完全性、継続取得可能性は保証しない。
- 通常練習と真剣練習は、実際の投資成績を保証するものではない。
