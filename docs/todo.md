## 次の本命: CSV読み込み・銘柄管理

有料向け機能を除くと、大きな未実装は以下。

1. ユーザーがブラウザから CSV ファイルを読み込む機能。
2. 読み込んだ CSV を銘柄として管理する機能。

実装前に、CSV・銘柄管理に関係する部分だけを最小限分割する。

### D-1. CSV・銘柄まわりの最小限ファイル分割

目的:

- App.tsx の肥大化を抑える。
- CSV 読み込み機能を安全に追加しやすくする。
- Codex の差分を小さくする。

実施済み:

- `src/types.ts`
- `src/data/defaultSymbols.ts`
- `src/utils/csv.ts`

保留:

- `src/utils/storage.ts`

保存処理は App 内の型や状態への依存が広いため、今回は無理に分割しない。

条件:

- UI変更はしない。
- 新機能はまだ追加しない。
- 既存の売買練習、チャートメモ、ペイント練習、本数計測は壊さない。
- `npm run build` と `npm run lint` が通ることを確認する。
- 変更が大きくなりそうなら、実装前に止めて確認する。

### D-2. ブラウザから CSV を一時読み込み

最初は保存なしでよい。

- ファイル選択で CSV を読み込む。
- `Date,Open,High,Low,Close` を検証する。
- `Volume` は任意。
- 読み込めたら一時銘柄としてチャート表示する。
- リロードしたら消えてよい。
- 既存の銘柄切り替えや売買練習を壊さない。

### D-3. 読み込んだ CSV の銘柄情報設定

設定項目:

- 銘柄名
- コード
- 市場 / 種別
- 通貨
- 1玉あたり数量
- multiplier
- 表示名

日本株以外の指数・為替・先物・仮想通貨でも使えるようにする。

### D-4. 銘柄管理

- 読み込んだ CSV を銘柄一覧に追加する。
- 銘柄一覧から選択できる。
- 名前変更できる。
- 削除できる。
- 再インポートできる。
- CSV データ本体は IndexedDB への保存を優先する。
- 銘柄メタ情報は localStorage または IndexedDB に保存する。

## Pro / 真剣練習向けに移動

### 練習結果サマリー

通常練習では急いで実装しない。

真剣練習モードとセットで実装する。

通常練習は、過去に戻れる・銘柄を変えられる・ログ削除できるため、厳密な成績サマリーとの相性が弱い。

真剣練習モードでは以下を制限する。

- 時間を戻せない。
- 途中で銘柄を変えられない。
- 売買ログを削除できない。
- 練習終了時に結果サマリーを表示する。
- 練習履歴を保存する。

サマリー表示候補:

- 勝ち数
- 負け数
- 引き分け数
- 総損益
- 最大利益
- 最大損失
- 売買回数
- 練習期間
- 勝率
- 平均利益
- 平均損失
- 最大ドローダウン
